# Executive Summary

## Inventory Management Database System

**Project Phoenix Repository #001**

---

## Overview

The **Inventory Management Database System** was developed in response to a simulated business case for **Bright Sparks Ltd**, a fictional Auckland-based electronics retailer. The objective of the project was to design and implement a relational database capable of improving inventory management, supplier administration and sales reporting through a structured and reliable data management solution.

The project demonstrates the practical application of database design principles by transforming business requirements into a fully functioning relational database. Developed using **MySQL** with a user interface built in **LibreOffice Base**, the system provides an efficient method for storing, managing and retrieving business information while supporting informed decision-making through queries, reports and data analysis.

---

## Business Context

Bright Sparks Ltd required a database solution capable of replacing manual inventory management processes with a centralised system that improved accuracy, efficiency and accessibility. The proposed solution was designed to manage product information, supplier records, inventory levels and customer sales while maintaining data integrity through the use of relational database principles.

The system also provides reporting functionality that enables management to identify low-stock products, monitor sales trends and evaluate inventory performance, allowing more informed purchasing and business decisions.

---

## Project Objectives

The primary objectives of this project were to:

- Design a relational database based on a realistic business scenario.
- Apply database normalisation techniques to improve data integrity.
- Develop a structured Entity Relationship Diagram (ERD).
- Create a functional MySQL database capable of storing and managing business information.
- Implement forms, queries and reports through LibreOffice Base.
- Provide role-based user permissions to improve data security.
- Demonstrate the complete software development lifecycle from planning through implementation and evaluation.

---

## Key Features

The completed system includes:

- Product Management
- Supplier Management
- Customer Management
- Sales Transaction Recording
- Inventory Tracking
- Weekly Low Stock Reporting
- Monthly Top Selling Product Reports
- SQL Queries and Reports
- User-Friendly Data Entry Forms
- Role-Based User Permissions

## Project Outcome

The Inventory Management Database System successfully met the objectives of the project by delivering a structured, scalable and user-friendly database solution capable of supporting the operational requirements of Bright Sparks Ltd.

Beyond the technical implementation, the project provided valuable experience in business analysis, database design, SQL development, documentation and software project management. It demonstrates the ability to translate business requirements into a practical software solution while following recognised database design principles and industry-standard development practices.
